# Streaming API

Endpoints for live video and audio streaming via Mux, DJ session management, video generation, and streaming webhook processing.

## List live streams

```http
GET /api/basefm/live
```

Returns all currently active Mux live streams. No authentication required.

### Response

```json
{
  "djs": [
    {
      "id": "aB1cD2eF3g",
      "name": "DJ Rave",
      "wallet": "0x1234...abcd",
      "playbackId": "xYz789",
      "streamKey": "sk-ab12-cd34-ef56",
      "status": "active",
      "startedAt": 1710806400,
      "hlsUrl": "https://stream.mux.com/xYz789.m3u8",
      "embedUrl": "https://stream.mux.com/xYz789.html"
    }
  ],
  "count": 1
}
```

| Field | Type | Description |
|-------|------|-------------|
| `djs[].id` | string | Mux live stream ID |
| `djs[].name` | string | DJ display name (defaults to `Anonymous DJ`) |
| `djs[].wallet` | string \| null | DJ wallet address from stream metadata |
| `djs[].playbackId` | string \| null | Mux playback ID |
| `djs[].streamKey` | string | RTMP stream key |
| `djs[].status` | string | Stream status (`active`) |
| `djs[].startedAt` | number | Unix timestamp when the stream was created |
| `djs[].hlsUrl` | string \| null | HLS playlist URL for playback |
| `djs[].embedUrl` | string \| null | Embeddable player URL |

### Errors

| Code | Description |
|------|-------------|
| 500 | Mux is not configured or an internal error occurred |

## Create live stream

```http
POST /api/basefm/streams
```

Creates a new Mux live stream. Access is granted if the caller meets either of the following conditions:

- The wallet holds at least **1,250,000 RAVE** tokens on the Base network, or
- The caller has an active **community guest pass** (available to Builder and Whale tier claimed holders via the community program)

<Note>The `wallet` address used for RAVE token verification must be a Coinbase Smart Wallet on Base. The platform enforces `smartWalletOnly` as its connector policy â€” injected wallets (such as MetaMask) are not supported for DJ streaming.</Note>

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `wallet` | string | Yes | Wallet address of the DJ. Used for RAVE token balance verification. |
| `name` | string | No | DJ display name (default: `Anonymous DJ`) |

### Response (200)

```json
{
  "success": true,
  "stream": {
    "id": "aB1cD2eF3g",
    "name": "DJ Rave",
    "wallet": "0x1234...abcd",
    "streamKey": "sk-ab12-cd34-ef56",
    "rtmpUrl": "rtmp://global-live.mux.com:5222/app",
    "fullRtmpUrl": "rtmp://global-live.mux.com:5222/app/sk-ab12-cd34-ef56",
    "playbackId": "xYz789",
    "status": "created",
    "accessGrantedBy": "rave"
  },
  "session": {
    "id": "sess_abc123",
    "maxDuration": 7200,
    "remaining": 7200,
    "expiresAt": "2026-04-11T02:00:00.000Z"
  },
  "obsSettings": {
    "server": "rtmp://global-live.mux.com:5222/app",
    "streamKey": "sk-ab12-cd34-ef56",
    "recommended": {
      "video": {
        "resolution": "1280x720 (720p) or 1920x1080 (1080p)",
        "bitrate": "2500-4500 kbps",
        "framerate": "30 fps",
        "encoder": "H.264",
        "keyframeInterval": "2 seconds"
      },
      "audio": {
        "bitrate": "256-320 kbps",
        "encoder": "AAC",
        "sampleRate": "44.1 kHz",
        "channels": "Stereo"
      }
    }
  },
  "streamType": "video+audio",
  "playback": {
    "hls": "https://stream.mux.com/xYz789.m3u8",
    "web": "https://stream.mux.com/xYz789.html"
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `stream.id` | string | Mux live stream ID |
| `stream.streamKey` | string | RTMP stream key |
| `stream.rtmpUrl` | string | RTMP ingest server URL |
| `stream.fullRtmpUrl` | string | Full RTMP URL including stream key (for OBS) |
| `stream.playbackId` | string \| null | Mux playback ID for viewers |
| `stream.accessGrantedBy` | string | How access was granted. Either `rave` (token balance) or `community-pass` (Builder/Whale community program perk). |
| `session.id` | string | DJ session record ID |
| `session.maxDuration` | number | Maximum session duration in seconds (7200 = 2 hours) |
| `session.remaining` | number | Remaining seconds in the session |
| `session.expiresAt` | string | ISO 8601 timestamp when the session expires |
| `obsSettings.recommended.video` | object | Recommended OBS video encoding settings |
| `obsSettings.recommended.video.resolution` | string | Recommended resolution (`1280x720` or `1920x1080`) |
| `obsSettings.recommended.video.bitrate` | string | Recommended video bitrate range |
| `obsSettings.recommended.video.framerate` | string | Recommended frame rate |
| `obsSettings.recommended.video.encoder` | string | Recommended video encoder (`H.264`) |
| `obsSettings.recommended.video.keyframeInterval` | string | Recommended keyframe interval |
| `obsSettings.recommended.audio` | object | Recommended OBS audio encoding settings |
| `streamType` | string | Type of stream. Currently always `video+audio`. |
| `playback.hls` | string \| null | HLS playlist URL for playback |
| `playback.web` | string \| null | Embeddable web player URL |

### Errors

| Code | Description |
|------|-------------|
| 400 | Wallet address required |
| 403 | Insufficient RAVE tokens or community guest pass. The wallet must hold at least 1,250,000 RAVE, or the authenticated user must have a Builder or Whale tier community claim. |
| 409 | An active session already exists for this wallet. The response includes a `remaining` field (seconds left) and the `sessionId`. |
| 500 | Mux is not configured or stream creation failed |

## Check session status

```http
GET /api/basefm/streams?wallet={wallet}
```

Returns the active DJ session for a wallet, including elapsed time and remaining duration.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `wallet` | string | Yes | Wallet address to check |

### Response â€” active session

```json
{
  "active": true,
  "session": {
    "id": "sess_abc123",
    "djName": "DJ Rave",
    "playbackId": "xYz789",
    "startedAt": "2026-04-11T00:00:00.000Z",
    "elapsed": 1800,
    "remaining": 5400,
    "remainingMinutes": 90,
    "expiresAt": "2026-04-11T02:00:00.000Z"
  }
}
```

### Response â€” no active session

```json
{
  "active": false,
  "message": "No active session. Start a new stream!"
}
```

If the session has expired (exceeded the 2-hour maximum), it is automatically ended and the response returns `active: false`.

| Field | Type | Description |
|-------|------|-------------|
| `active` | boolean | Whether the wallet has an active session |
| `session.id` | string | Session record ID |
| `session.djName` | string | DJ display name |
| `session.playbackId` | string \| null | Mux playback ID |
| `session.startedAt` | string | ISO 8601 timestamp when the session started |
| `session.elapsed` | number | Seconds elapsed since the session started |
| `session.remaining` | number | Seconds remaining before the session expires |
| `session.remainingMinutes` | number | Minutes remaining (rounded down) |
| `session.expiresAt` | string | ISO 8601 timestamp when the session expires |

### Errors

| Code | Description |
|------|-------------|
| 400 | Wallet query parameter is required |

## End session

```http
DELETE /api/basefm/streams?wallet={wallet}
```

Ends any active DJ session for the given wallet.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `wallet` | string | Yes | Wallet address whose session should be ended |

### Response

```json
{
  "success": true,
  "ended": 1,
  "message": "Session ended"
}
```

If no active session exists, `ended` is `0` and `message` is `"No active session found"`.

| Field | Type | Description |
|-------|------|-------------|
| `success` | boolean | Always `true` |
| `ended` | number | Number of sessions ended (0 or 1) |
| `message` | string | Human-readable status message |

### Errors

| Code | Description |
|------|-------------|
| 400 | Wallet query parameter is required |

## Generate video

```http
POST /api/generate-video
```

Generates a video and uploads it to blob storage. Requires session authentication. This endpoint has a 5-minute timeout.

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Video type. One of: `demo`, `marketing`, `screenshot`, `tutorial`. |
| `agentName` | string | For `demo` | Agent name for the demo video |
| `agentDescription` | string | For `demo` | Agent description for the demo video |
| `productName` | string | For `marketing` | Product name for the marketing video |
| `features` | any | For `marketing` | Product features for the marketing video |
| `imageUrl` | string | For `screenshot` | Image URL to animate |
| `description` | string | For `screenshot` | Description for the animation |
| `topic` | string | For `tutorial` | Tutorial topic |
| `steps` | any | For `tutorial` | Tutorial steps |

### Response

```json
{
  "url": "https://blob-storage-url/videos/1710806400000.mp4"
}
```

### Errors

| Code | Description |
|------|-------------|
| 400 | Invalid video type |
| 401 | Unauthorized â€” session required |
| 500 | Video generation or upload failed |

## Mux webhooks

```http
POST /api/webhooks/mux
```

Receives and processes Mux webhook events. This endpoint verifies the request signature using HMAC-SHA256 and rejects unsigned, expired, or tampered requests.

<Warning>This endpoint is intended to be called by Mux only. You must configure the `MUX_SIGNING_SECRET` (or `MUX_WEBHOOK_SECRET`) environment variable for signature verification. When the signing secret is not configured, all requests are rejected.</Warning>

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `mux-signature` | Yes | Mux webhook signature in the format `t=<timestamp>,v1=<signature>` |

### Signature verification

The endpoint performs the following checks:

1. Rejects requests missing the `mux-signature` header (`401`)
2. Rejects requests with a timestamp older than 5 minutes to prevent replay attacks (`403`)
3. Computes HMAC-SHA256 over `<timestamp>.<body>` using the signing secret
4. Performs a timing-safe comparison of the computed signature against the provided signature (`403` on mismatch)

### Handled event types

| Event | Behavior |
|-------|----------|
| `video.asset.ready` | Archives HD assets (1080p or higher) longer than 15 minutes. Shorter or lower-resolution assets are queued for deletion. |
| `video.live_stream.active` | Logs that a live stream has become active |
| `video.live_stream.idle` | Logs that a live stream has stopped |

### Response

```json
{
  "received": true
}
```

### Errors

| Code | Description |
|------|-------------|
| 401 | Missing `mux-signature` header |
| 403 | Invalid signature or expired timestamp |
| 500 | Webhook processing failed |
