# Admin Railway Orphans API

List all users that have an active Railway (OpenClaw) instance. Use this endpoint to audit provisioned instances and detect orphaned resources that may no longer be needed.

## List users with Railway instances

```http
GET /api/admin/railway-orphans
```

Returns every user whose `openclawInstanceId` is not null, along with key account and instance details.

### Authentication

Requires a valid `INTERNAL_API_KEY` passed as a Bearer token. Requests without a valid key receive a `401` response.

```http
Authorization: Bearer <INTERNAL_API_KEY>
```

### Response

```json
{
  "users": [
    {
      "id": "user_abc123",
      "email": "user@example.com",
      "plan": "pro",
      "subscriptionStatus": "active",
      "openclawInstanceId": "inst_xyz789",
      "openclawUrl": "https://instance.up.railway.app"
    }
  ]
}
```

### Response fields

| Field | Type | Description |
|-------|------|-------------|
| `users` | array | List of users with a provisioned Railway instance |
| `users[].id` | string | User ID |
| `users[].email` | string | User email address |
| `users[].plan` | string | Current subscription plan (for example `free`, `pro`) |
| `users[].subscriptionStatus` | string | Subscription status (for example `active`, `canceled`, `past_due`) |
| `users[].openclawInstanceId` | string | Railway instance identifier |
| `users[].openclawUrl` | string | Public URL of the Railway instance |

### Errors

| Code | Description |
|------|-------------|
| 401 | Unauthorized — missing or invalid `INTERNAL_API_KEY` |
